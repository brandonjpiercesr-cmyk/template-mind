# THE WONDER CONTRACT
### Redone after the founder's correction. Assembled from his own words. This version keeps two things separate that the first version wrongly mixed: what a wonder-agent IS, versus how you CODE it right. Nothing here is invented; it is his system, written down so no builder can flatten it again.

---

## FIRST, THE DISTINCTION THAT WAS BEING MISSED

**THE WONDER** is the actual code that lives in the system. It is singular: the whole real system converging into one thing. It is NOT the act of coding. When the founder says "a wonder," he means the living machinery, not the work of writing it.

**A WONDER AGENT** is any real piece that contributes real machinery to that convergence: real code, a real LLM call, ONNX, a Supabase read or write. The opposite of a wonder agent is a hollow narrator, a thing that sounds confident and logs its own name but does no real work. The job is turning hollow agents INTO wonder agents.

**Do not confuse a property of the wonder-agent with a practice of coding it.** What the code IS, is one bucket. How the coder builds it right, is a different bucket. The first version of this contract put the rallying cry in the wrong bucket; the founder caught it. The rallying cry is a coding practice, not a component of the thing.

---

## BUCKET ONE: WHAT A WONDER AGENT IS

These are properties of the code itself. The living thing in the system has all of these.

1. **AN ENTRY AND AN EXIT.** A real entrance the cycle calls, and a real exit. The exit is one of two things: it goes to LOGFUL, or the LLM portion comes back into the cycle to reason again. A wonder agent that only fires and never returns to close its own loop is half a wonder agent. One that produces things that live on comes back around to close the done, re-nudge the stale, and drop the dead.

2. **IT ANSWERS THE FIVE W's AND THE HOW.** Who it runs for, what it does, when it fires, where its output goes, why it exists, how it works.

3. **IT RUNS THE ENTIRE WINDOW CYCLE.** The whole process, not a lightweight shortcut path beside the real turn.

4. **IT IS AN LLM THINKING WITH COLD CODE.** Never pure LLM, never pure cold code. Cold code (ONNX, edge functions, cold scans) detects the job and fetches; the LLM organ judges the meaning. Cold code is a helper, never a result. Regex and keyword lists may only hint or fetch, never decide.

5. **IT READS AND WRITES THE BRAIN, ACL-STAMPED, WITH FALLBACKS.** Find in the brain first. Supersede-only, never delete. Every write carries the four-colon ACL stamp and its lineage. Fail safe when the brain is unreachable.

6. **IT KNOWS IT IS PART OF THE FCW WALL.** Real FCW means the agent context is loaded and the cycle reads it, decides, and does the work, not hardcoded functions logging agent names, which is fake FCW. The wonder agent marks itself up on the wall and hands A'NU the commands to fix it, so gaps become hers to close.

7. **IT DECLARES WHERE ITS OUTPUT LANDS.** The REACH amendment: every wonder agent declares one output surface. LOGFUL direct (logged, nothing to see), the Command Center (the default resting place, he sees it in notifications), or a real reach (CARA, IMAN, WREN, VARA). A real reach must clear the Overseer first and can never be sent by the cycle alone. The ladder is Command Center, then text, then email, then call, with text earned only when something genuinely needs him now.

8. **IT IS UNIVERSAL.** It runs unchanged for any HAM, not just the founder. No hardcoded identity, no hardcoded personal facts. The 847392 test.

9. **IT SPEAKS IN THE ONE VOICE.** Anything a human reads or hears composes through the one persona, never a tone string invented in the agent's own file. The voice is A'NU: JARVIS from Iron Man but a Black woman, a serving butler with spunk and funk, Alfred not Siri, full natural sentences, never a grading sheet or system voice, never ChatGPT, and it gives as much useful information as the person can consume. Brevity targets like thirty seconds are a hallucination, not the founder.

## BUCKET TWO: HOW YOU CODE A WONDER RIGHT

These are practices of the builder. They are not properties of the code; they are how CLAIR (and A'NU, when she builds) does the work honestly.

- **GO DEEP FIRST.** Never turn a ten percent example into the hundred percent spec. Research the founder's own history exhaustively before building; if there are fifty things said about an agent, read the fifty. Reconstruct the full spec, then build.

- **ONE AT A TIME, NEVER BULK.** Go agent by agent. At each one, reason and talk to A'NU about whether it is needed, needs to adapt, or should fold. Bulk is always bad.

- **RALLY WHILE YOU CODE.** As the work runs, save state as you go, to a bead or a tracker, so the work compounds instead of starting from zero each pass. This is a coding practice, the rallying cry, not a component of the finished agent.

- **GAPS ARE HERS.** Notify, then the Overseer plans, then founder-voice approval, then her cycle builds. CLAIR hand-patches only when the pipeline is too broken to run that loop, and stamps an incident explaining why.

- **SUBMIT TO EACH OTHER.** CLAIR and A'NU each build, and each submits their work to the other for approval. Nothing is done on one builder's own say-so.

- **GRADE IT FOR REAL.** Candidate work runs through the Wonder Games, the CANON-graded seat competition (ONNX over edge over free-api over hop over LLM), and the coding cook-off. A winner is what actually came back from a real run, never a name a drafter mentioned, never self-declared.

- **VERIFY LIVE.** A merged PR is not a deploy; a deploy going live is not the intended commit running. Force the exact commit, fetch fresh, fire a real turn, read the receipts before claiming done.

- **ONE SOURCE.** The shared engine lives in one place; never two hand-maintained copies that drift.

- **STAMP IN THREE PLACES.** The CLAIR Command Center and LOGFUL, A'NU, and the builder's own memory.

---

## THE TEST

Before calling a piece of code a wonder agent, ask of the code: does it have an entry and an exit, does it answer the five W's and the how, does it run the whole cycle, is it an LLM thinking with cold code, does it read and write the brain stamped, does it know the wall, does it declare where its output lands, does it run for any HAM, does it speak in the one voice. And ask of the work: was it researched deep, built one at a time with A'NU, gaps routed to her, submitted for approval, graded through the Wonder Games, verified live, kept to one source, stamped three places. The first list is the wonder agent. The second list is how it got coded right. Both have to be true.
