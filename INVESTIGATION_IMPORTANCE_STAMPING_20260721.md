## THE VERDICT IN ONE PARAGRAPH
Your instinct is correct in spirit and nearly correct in letter. Importance-scoring has an honest lineage — Stanford's Generative Agents (2023) scored memories as **recency + importance + relevance**, summed — but that system used importance as one of three *added* terms, with the relevance term (embedding similarity against the current question) as the backbone. Your system kept recency and importance, made importance a **hard gate**, hardcoded the scores as per-call-site constants, and **skipped the relevance leg entirely**. That configuration is standard nowhere. No production memory system (MemGPT/Letta, LangGraph/LangMem, Supabase's own documented patterns, modern hybrid-RAG) runs importance+recency without a relevance leg; relevance is the backbone everywhere, importance at most a modifier. And the LLM-judge literature documents exactly the score-inflation you smelled: scores compress toward the high end systematically.

## WHAT THE CODE ACTUALLY DOES
- **319 hardcoded `importance:` literals.** Distribution: mode at 6–7, 68% in the 5–8 band. Not literally everything-is-a-9 — but the scores are **coder-chosen constants per file**, not per-content judgments: every draft is an 8, every deploy error a 10, every ghostwriter output a 6. The 1–10 scale is functionally 3–4 event-type buckets carrying no per-item information.
- **Only two places let a mind assign importance;** everywhere else it's the cold-code reflex — the score is instant proof-of-work requiring no judgment at read time. It is precisely the reach-down-to-cold-code pattern the wonder-first law names.
- **Inconsistent defaults:** `core/brain.client.js:77` defaults importance to **0** while `agents/logful.js:23` defaults to **5** — beads written without an explicit score are invisible to every ≥7/≥8 finder. Silent memory loss.
- **Both times salience actually mattered, importance failed:** the ventilation flood (machinery drowned conversation — fixed by source-prefix, not importance) and the 139-missed-calls incident (machinery stamped ≥8 flooded the founder-alert pool — fixed by stamp_type exclusion). Each real fix migrated the discriminative work to `stamp_type`/`source`. Importance survives mostly as ballast.
- **The buried lede — the relevance leg was faked once and orphaned once.** `substrate/tim.js` already downloads and runs a real embedding model (MiniLM, sub-millisecond, in-process) but was never pointed at the brain. And `core/tim.wiring.js` records that a prior coder shipped a **fake hash-based "embedding" explicitly commented 'Simple hash-based embedding for demo'** with zero callers. That is puddy, verbatim: scaffold that looks like the organ and holds no weight.

## WHAT REPLACES IT (Phase 2 spec, on your existing stack)
1. **Give the brain a real relevance leg.** `embedding vector(384)` column on `aibe_brain` (matches the MiniLM already running in-process), backfilled from summary+content, HNSW index. Supabase supports all of this natively (pgvector + the documented `match_documents` RPC pattern; optionally hybrid keyword+vector with RRF).
2. **A `match_beads` RPC** — semantic match with your existing walls as WHERE clauses (ham_uid, stamp_type, acl) so per-HAM isolation holds inside the search itself.
3. **The retrieval wonder.** Cold code fetches top-K candidates by meaning; a deliberating mind reads them against the actual question and decides the strand — with a logged why. This is not a concession to the wonder-first philosophy; it is what the industry calls the reranking stage, done as a thinking partner leashed to traceable cold code. (It is also the Keeper's retrieval spine — Phases 1 and 2 share this organ.)
4. **Demote importance; don't delete it.** Keep it as a tie-breaker/decay weight inside ranking (the original Stanford usage — one normalized additive term, never a gate). Keep the one honest use found (the ≥4 person-facing screen rail — genuinely noise-vs-signal for a human surface). Fix the write side: a short honest rubric (10 = founder doctrine/irreversible, 7 = decisions, 5 = routine output, 2 = machinery), one consistent default, and where a wonder writes a bead, the wonder assigns the score with a one-line rationale — calibrated against the documented LLM tendency to inflate.
